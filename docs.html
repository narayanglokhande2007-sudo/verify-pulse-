Introduction

Welcome to the VerifyPulse Enterprise API. Our API provides a hyper-scalable, zero-day threat detection engine specifically built for Indian Digital Payments.

API Base URL (Edge Network): https://verify-pulse.vercel.app

Architecture: How We Defeat LLM Hallucinations

Unlike standard AI wrappers, VerifyPulse does not rely on continuous LLM fine-tuning (which suffers from 90-day memory decay). Instead, we use a proprietary 4-Layer Engine:

  • Layer 1 (Immutable Cache): A hardcoded static whitelist of 200+ verified Indian financial domains. Safe URLs resolve locally in <50ms without ever hitting the LLM.
  • Layer 2 (Real-Time RAG): We pull live threat intelligence from 20+ global data sources (PhishTank, UrlHaus) into a latest_scams.json pipeline. Our Edge Functions dynamically inject this live data into the Llama-3 prompt via Retrieval-Augmented Generation (RAG).
  • Layer 3 (Llama 3 70B AI): The AI analyzes the payload contextually using the injected live-cheat-sheet, guaranteeing zero-day detection without continuous model training.
  • Layer 4 (Multimodal Vision): For advanced threats like fake payment screenshots or AI voice-phishing (deepfakes), our engine seamlessly routes binary payloads to a secondary Gemini Vision pipeline for deep multimodal inspection.

Authentication

We process all requests statelessly. To identify your enterprise account, pass an API Key via the standard Authorization Bearer header.

Note: If you are testing the live playground without an account, the system will accept requests without a token in 'Sandbox Mode'.

Analyze Payload

POST /api/verify

This is the primary endpoint to scan any text or URL for malicious intent.

Request Parameters

Parameter Type Description
text string Required. The raw message or URL to be analyzed.
checkType string Required. Must be exactly scan.

Error Handling

VerifyPulse uses standard HTTP status codes to indicate the success or failure of a request.

Status Code Description
200 OK The payload was successfully analyzed.
400 Bad Request Missing the text or checkType parameters.
500 Internal Error The Edge network timed out. Fallback safely to your app.
Example Request (cURL)
curl -X POST https://verify-pulse.vercel.app/api/verify \
  -H "Authorization: Bearer vp_live_xxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Your SBI KYC is pending. Update here: http://sbi-update.in",
    "checkType": "scan"
  }'
